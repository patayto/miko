A first-contact journal entry capturing early impressions and feature discovery after watching YouTube tutorials on [[LogSeq]]. The entry is exploratory — a live note taken while learning the tool.

The central workflow idea is using the daily journal as an [[Interstitial Journaling]] surface: timestamped entries written throughout the day without needing to decide where each note belongs upfront. The appeal is that LogSeq's linking and query system handles organization retroactively, reducing the friction of structured filing.

Several UI features stood out. Shift-clicking an embedded block opens it side-by-side — useful for referencing while writing. The `/time` command inserts a timestamp, which pairs naturally with interstitial journaling and meeting notes. The `/date` command links to a future date page, so any block referencing that date will surface there — a lightweight reminder mechanism. [[LogSeq Slash Commands]] are flagged as a rich area worth deeper research later.

For multiline content, the `#+BEGIN_TIP ... #+END_TIP` advanced command (triggered via `<`) solves a concern about block-level writing constraints. The entry also celebrates embedded and rendered HTML (via `@@html:@@` syntax) and embedded YouTube videos with timestamped playback — both treated as delightful discoveries.

Project management within LogSeq gets a critical look. A kanban view would be desirable, but native support does not exist. One community plugin exists (`logseq-kanban-plugin`) but is assessed as weak: it cannot create columns or tasks within the renderer. Whether to build a custom plugin is floated as an open question.

Syncing is noted as an unresolved choice: cloud storage (e.g. iCloud/Dropbox), the paid LogSeq Sync beta, or a self-hosted cloud vault. The self-hosted option is preferred on privacy grounds.

Integrations mentioned include Readwise.
