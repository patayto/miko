A comprehensive interview preparation guide for [[Deliveroo]]'s technical assessment. The assessment is divided into two sections: a 30-minute multiple-choice section covering [[REST API Design Fundamentals]], [[Systems Design]], and [[Testing Fundamentals]], followed by a 70-minute live coding section with two medium-difficulty problems.

The first section focuses on REST API concepts including HTTP methods, status codes, pagination, query vs path parameters, and authentication basics. Systems design coverage emphasizes statelessness, caching strategies, scalability, REST principles, and idempotency. The testing section covers the test pyramid model (unit tests as the base, integration tests in the middle, E2E tests at the top), functional vs non-functional testing, contract testing, and the AAA pattern for test organization.

For coding practice, the guide recommends using [[HackerRank]] as the primary platform, focusing on medium-difficulty problems in data structures and algorithms. Key areas include arrays, linked lists, trees, stacks, queues, hash tables, and algorithm implementation. The guide suggests a six-day study plan: two days for REST API and arrays, two days for systems design and hash maps/trees, one day for testing and mixed problems, and one day for a full mock test.

The prep strategy emphasizes time management: one minute per multiple-choice question with flagging of uncertain answers for later review, and 35 minutes per coding problem (5 minutes reading/planning, 25 minutes coding, 5 minutes testing). The guide recommends Python or JavaScript/Node.js for the assessment, with native library usage and no external frameworks. UI familiarization with [[HackerRank]] is recommended before the assessment, including practice with their code editor and the Run vs Submit workflow.
